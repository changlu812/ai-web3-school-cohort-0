## 核心知识点

### Instruction

- **节点**: Instruction
- **要点**: 四段式写法：任务目标、可用输入、禁止行为、输出格式

### Few-shot

- **节点**: Few-shot
- **要点**: 好示例+坏示例引导风格，但需跟 eval 一起维护

### Structured Output

- **节点**: Structured Output
- **要点**: JSON schema 让机器能校验、拒绝、回归测试

### Prompt Injection

- **节点**: Prompt Injection
- **要点**: 不靠 prompt 防注入，靠标记不可信数据+参数校验+人工审批

## 最小实践

设计了 **"交易风险摘要" Prompt**，要求输出固定 JSON 格式：

```json
{
  "summary": "交易摘要描述",
  "asset_changes": [
    {
      "asset": "资产名称",
      "change": "变化量"
    }
  ],
  "risk_level": "low|medium|high",
  "requires_human_approval": true|false
}
```

并准备三组测试用例用于回归测试。

---

## 学习心得

1. Prompt Engineering 不是玄学，而是工程化方法
2. 结构化输出（JSON Schema）是 AI × Web3 的关键桥梁
3. 安全性不能依赖 prompt 本身，需要多层防御
4. Few-shot 示例需要持续维护，与 eval 体系结合
